# 1️⃣ 개념 정리 

## 10. 데이터 탐색과 시각화

```
✅ 학습 목표 :
* EDA의 목적을 설명할 수 있다.
* 주어진 데이터셋에서 이상치, 누락값, 분포 등을 식별하고 EDA 결과를 바탕으로 데이터셋의 특징을 해석할 수 있다.
* 공분산과 상관계수를 활용하여 두 변수 간의 관계를 해석할 수 있다.
* 적절한 시각화 기법을 선택하여 데이터의 특성을 효과적으로 전달할 수 있다.
```

---
**<분포 시각화>** 

- 분포 시각화는 데이터의 구성과 비율을 직관적으로 이해하기 위한 기본적이지만 중요한 방법임
- 통계치만으로는 한계가 있으므로, 시각화를 통해 변수의 분포와 구조를 쉽게 파악할 수 있음

👉 연속형(양적 척도):
- 막대그래프, 선그래프, 히스토그램 사용
- 히스토그램은 데이터를 일정한 간격(bin)으로 나누어 구간별 도수를 막대로 표현
- 구간 수는 처음엔 세분화(약 20개) 후, 시각적 손실이 크지 않은 범위에서 줄이는 것이 좋음
- 구간이 너무 많으면 복잡, 너무 적으면 정보 손실 발생

👉 명목형(질적 척도):
- 파이차트, 도넛차트 사용
- 전체를 100%로 보고 각 구성 요소의 비율을 면적(각도)으로 표시
- 비율 수치를 함께 표기하면 가독성 향상
- 도넛차트는 가운데 공간을 활용해 전체 값 등 추가 정보 삽입 가능

👉 복잡한 질적 척도:
- 트리맵 차트 사용 — 큰 사각형을 비율에 따라 작은 사각형으로 분할
- 위계 구조 표현 가능 (예: ‘바지’ → ‘긴 바지’·‘반바지’)
- 단점: 구성 요소 간 규모 차이 크면 시각화 어려움
- 와플 차트는 트리맵과 유사하나 위계 구조 표현 불가, 균등한 격자 형태로 단순 비율만 표시

💡요약 
* 연속형 데이터 → 히스토그램 중심,
* 단순 명목형 → 파이/도넛차트,
* 복잡 명목형 → 트리맵(또는 와플)
---
**<관계 시각화>** 
- 공분산·상관분석을 기반으로 변수 간 관계를 시각화할 때 주로 산점도와 버블차트를 사용함

👉 산점도(Scatter Plot)
- 두 연속형 변수 간 관계를 가장 단순하게 표현하는 방법
- X축, Y축에 각각 변수를 두고 일치 지점에 점을 찍음
- 점들의 분포·추세를 통해 관계 방향 및 강도 파악 가능
- 극단치 제거 필요 — 주요 분포 구간이 압축되어 시각적 효율 저하 방지
- 데이터가 많을 경우 투명도 조정 또는 색상 농도 표현으로 점의 밀도 시각화 가능
- Python, R 등에서 쉽게 구현 가능

👉 버블차트(Bubble Chart)
- 산점도의 확장형으로, 버블 크기를 통해 세 번째 변수 표현 가능
- 색상 요소를 추가해 네 번째 변수까지 구분 가능하지만, 정보 과부하 주의
- 색상은 그룹 구분용으로만 사용하는 것이 바람직
- 애니메이션 추가 시 시간에 따른 변화도 표현 가능(다만 구현은 복잡)
- 관측치가 100개 이상이면 정보 전달력 떨어짐 → 데이터 축약 또는 다른 시각화 권장
- 해석 시 원의 면적으로 크기 판단해야 함 (지름 두 배 → 면적 네 배)

💡요약 
- 산점도 → 2변수 관계 단순 시각화 (극단치·투명도 고려)
- 버블차트 → 3~4변수 표현 가능하나 과도한 정보는 피해야 함

---
**<공간 시각화>** 
* 공간 시각화(Spatial Visualization)는 데이터에 지리적 위치 정보(위도·경도)를 결합해 지도 위에 표현하는 기법임
* 지도 기반 시각화는 단순 표나 그래프보다 데이터의 분포·패턴을 직관적이고 명확하게 보여줌

💡특징
* 위도·경도 또는 지명만으로 위치 표현 가능 (예: Google GeoMap)
* 단순 이미지가 아닌 확대·이동 등 인터랙티브 기능 제공
* 분석 시 거시적→미시적 스토리라인을 고려하면 전달 효과 극대화

👉 도트맵(Dot Map)
* 지도에 동일한 크기의 점으로 데이터 분포 표시
* 점 하나는 개별 단위나 특정 수량을 의미
* 분포 개요 파악엔 유리하지만 정확한 수치 표현엔 부적합
* 확대 시 숫자, 축소 시 점으로 표현하는 하이브리드 방식도 존재

👉 버블맵(Bubble Map)
* 버블차트를 지도 위에 구현한 형태
* 버블 크기로 데이터 값 표시 → 비율 비교 용이
* 단점: 버블이 겹칠 수 있어 크기 조정 필요

👉 코로플레스맵(Choropleth Map)
* 지역별 데이터 값을 색상 음영으로 표현 (단계 구분도)
* 색상·명도·채도로 값의 차이 표시 가능
* 정확한 수치 비교는 어려움, 큰 지역이 과도하게 강조될 수 있음

👉 커넥션맵(Connection/Link Map)
* 지도상의 점들을 **선(곡선·직선)**으로 연결하여 지리적 관계 표현
* 경로·분포·집중도 시각화에 활용
* 예시: 무역 관계, 항공 노선, 통신망 등

👉 기타 기법
* 프로우맵(Flow Map): 선의 방향·굵기로 시작점~도착점의 흐름 표현
* 카토그램(Cartogram): 지역 면적을 데이터 값에 비례하도록 변형
* 지도 위에 바 차트·파이차트를 결합해 지역별 세부 비교도 가능

---
**<박스 플롯>**
* 박스플롯은 하나의 그림으로 양적 데이터의 분포, 중심, 편향성, 이상치 등을 한눈에 보여주는 시각화 기법임
* 특히 두 변수의 분포를 비교하거나 카테고리별 분포를 분석할 때 효과적임

👉 주요 특징 
* 데이터의 전반적 분포 형태를 간결하게 표현
* 중앙값, 사분위수, 이상치 등 5가지 핵심 통계값을 시각화
* 분석 시 데이터의 대칭성·편향성·이상치 존재 여부를 빠르게 파악 가능

👉 5가지 대표 수치 

(1) 최솟값 (Min): Q1 - 1.5×IQR

(2) 제1사분위수 (Q1): 하위 25% 위치

(3) 제2사분위수 (Q2): 중앙값 (50%)

(4) 제3사분위수 (Q3): 상위 75% 위치

(5) 최댓값 (Max): Q3 + 1.5×IQR
※ IQR(Interquartile Range) = Q3 - Q1
※ 평균값(mean)도 함께 표시 가능
※ 범위를 벗어난 값은 **이상치(outlier)**로 점 표시

👉 해석 방법 
* 박스(네모) = 전체 데이터의 중간 50%
* 박스 안의 선 = 중앙값 (Q2)
* 수염(Whisker) = 이상치를 제외한 최댓값·최솟값 범위
* 수염 밖의 점 = 이상치
* 박스 길이 짧음 → 데이터 집중 / 길면 분산 큼

![(참고 그림) ](<images/WEEK_5_실습/박스플롯 참고자료.png>)

👉 비교와 분포 해석 
* 여러 변수의 박스플롯을 나란히 배치해 분포 차이 비교 가능
* 박스(50%)는 정규분포의 ±1표준편차 구간과 유사
* 수염 전체 범위는 ±3표준편차와 유사
* 해석 시 항상 **데이터의 실제 분포 형태를 함께 고려해야 함**
