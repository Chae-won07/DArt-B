### 11.1. 결측값 처리

👉 결측값(missing value)이나 이상치(outlier)는 실제 분석 데이터에서 자주 발생하며, 탐색 단계에서 이를 파악하고 적절히 처리하는 과정이 필수적임.

👉 결측값의 유형 
* (1) 완전 무작위 결측(MCAR: Missing Completely at Random) 
  - 결측이 완전히 무작위로 발생 
  - 데이터를 제거해도 편향이 거의 없음 
* (2) 무작위 결측(MAR: Missing at Random)
  - 결측이 해당 변수가 아닌 다른 변수의 특성에 의해 체계적으로 발생 
* (3) 비무작위 결측(NMAR: Not Missing at Random)
  - 결측이 해당 변수 자체의 특성에 의해 발생 

👉 결측값 처리 방법
* (1) 제거(Removal)
  - 결측값이 심하게 많은 변수나 관측치를 제거하는 방법 
  - 전체 데이터의 결측 비율이 1% 미만이라면 무시해도 무방 
  - 단, 대부분의 데이터에는 결함이 있으므로, 처리 전에 반드시 결측 비율을 파악해야 함 

* (2) 대치(Imputation)
  - ① 평균대치(Mean Imputaion) : 결측값을 관측값의 평균으로 대체하는 가장 단순한 방법 → 간편하지만 표준오차 축소, p-value 부정확성, 편향 증가의 단점 존재
  - ② 보간법(Interpolation) : 시계열 데이터에서 사용하며, 결측값을 전·후 시점의 값 또는 평균값으로 대체
  - ③ 회귀 대치(Regression Imputation) : 다른 변수들과의 관계를 고려하여 회귀 분석으로 예측값을 대치 → 단순 평균 대치의 단점을 보완하는 합리적 방법
  - ➃ 다중 대치(Multiple Imputation) : 단순 대치법의 오차분산 과소 추정을 해결하기 위한 방법으로, 가상의 데이터셋 여러 개를 생성하고, 각각의 분석 결과를 결합하여 최종 대치값을 산출함 
    - 세 단계로 구성됨: 대치(Imputation) → 분석(Analysis) → 결합(Pooling)


### 11.2. 이상치 처리

👉 이상치란 일부 관측값이 전체 데이터의 범위에서 크게 벗어난
극단적으로 작거나 큰 값을 의미함

👉 이상치의 문제점
- 데이터의 모집단 평균·총합 추정에 왜곡 발생
- 분산 증가, 모델 정확도 저하 초래
- 특히 회귀분석 시 회귀선 경사 왜곡으로 예측력 약화 가능

👉 이상치 처리 방법
* (1) 이상치 제거 (Trimming)
  - 해당 관측치를 아예 제거
  - → 단순하고 직관적이지만,
데이터 양이 충분할 경우 평균에 미치는 영향이 작아 제거 필요성 낮아질 수 있음
* (2) 값 대치 (Value Modification)
  - 이상치를 상한값/하한값으로 대체하여 영향 완화
  - 가중치 조정(weight modification)으로도 영향 줄이기 가능

👉 이상치 식별 기준
* EDA(탐색적 데이터 분석), 시각화, 박스플롯(Box Plot) 등을 이용해 탐색
* ① **박스 플롯 기반**:
IQR(사분위 범위)의 1.5배를 벗어나는 값을 이상치로 간주
* ② **통계적 기준**:
평균(또는 중앙값)으로부터 ±n 표준편차 이상 떨어진 값
* ③ **MAD** (Median Absolute Deviation):
평균보다 이상치에 덜 민감한 중앙값 기반 측정치로 이상치 탐지에 효과적
* ➃ **도메인 지식**:
변수의 의미나 비즈니스 맥락을 고려해 이상치 원인을 논리적으로 파악
예: 제조 데이터에서 상·하한선을 벗어난 센서값 → 불량 가능성 분석

### 11.3. 변수 구간화

👉 변수 구간화(Binning)는 분석 성능 향상과 해석의 편리성을 위해
연속형 변수를 범주형 변수로 변환하는 기법임

👉 데이터의 패턴을 단순화하고, 모델의 안정성과 예측력을 높이는 데 활용됨

👉 구간화 방법
* (1) 동일 폭 (Equal Width)
  - 변수의 최솟값과 최댓값을 일정한 간격으로 나누는 방법
  - 단순하지만, 데이터가 한쪽으로 치우쳐 있으면 불균형 발생 가능
* (2) 동일 빈도 (Equal Frequency)
  - 각 구간에 속하는 관측치의 개수가 유사하도록 나누는 방법
  - 데이터 분포의 왜곡을 줄이는 데 효과적
* (3) 평활화 (Smoothing)
  - 구간화된 값을 해당 구간의 평균·중앙값·경계값으로 대체
  - 변수의 미세한 변동성을 완화하는 단순한 방법

👉 효과적인 구간화 기준
* 구간화의 적절성을 판단하기 위해 WOE (Weight of Evidence)와 IV (Information Value)를 활용

💡IV (Information Value)
* 종속변수의 True/False 구분 능력을 나타내는 지표
* IV 값이 높을수록 **예측력(설명력)**이 높은 변수로 평가됨
  - IV < 0.02: 예측력 매우 약함
  - 0.02 ≤ IV < 0.1: 약함 (Weak)
  - 0.1 ≤ IV < 0.3: 보통 (Medium)
  - IV ≥ 0.3: 우수함 (Strong)

👉 파이썬·R 등 분석 환경에서 IV를 이용해 효과적 구간화 수행 가능

### 11.4. 데이터 표준화와 정규화 스케일링

👉 **표준화(Standardization)**와 **정규화(Normalization)**는
서로 다른 단위·편차를 가진 독립 변수들의 스케일을 일정하게 조정하는 기법임

👉 데이터 분석 결과의 안정성과 머신러닝 모델의 학습 효율을 높이는 데 활용됨

👉 특히 k-NN, SVM 등 거리 기반 모델에서는 필수적으로 수행되어야 함

👉 표준화 (Standardization)
* 개념:
각 관측치가 평균으로부터 얼마나 떨어져 있는지를 나타내는 변환
* 특징:
  - 평균(Mean)을 0, 표준편차(Standard Deviation)를 1로 변환
  - 변환된 값은 Z-score로 표현 (±1, ±2 등 표준편차 단위로 표시)

👉 정규화 (Normalization)
* 개념:
데이터 값을 0과 1 사이의 범위로 변환하여 분포를 조정하는 기법
* 특징:
  - 최솟값은 0, 최댓값은 1
  - 직관적이며, 스케일 차이로 인한 왜곡을 방지

👉 RobustScaler
* 개념:
이상치(outlier)에 민감한 표준화·정규화의 단점을 보완한 스케일링 방법
* 특징: 
  - 중앙값(Q2)을 0으로 설정
  - Q1(25%) ~ Q3(75%)의 IQR을 1로 조정
  - 평균 대신 중앙값 사용 → 극단값(이상치)의 영향 최소화
  - 실제 데이터 분포가 왜곡된 경우에도 안정적인 성능 제공
